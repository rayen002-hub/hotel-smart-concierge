\section*{Introduction}

Ce chapitre détaille la troisième et dernière release du projet. Elle couvre le workflow complet des réclamations avec classification automatique par intelligence artificielle, l'application Flutter mobile pour les agents d'intervention terrain, et le module de gestion des tâches de ménage de chambre (housekeeping). L'intelligence artificielle (classification par apprentissage automatique) est intégrée directement dans le Sprint 7, et non traitée comme une release séparée.

\subsection{Étude conceptuelle}

\subsubsection{Modèle de données}

Le modèle \texttt{Complaint} comprend les champs principaux : \texttt{originalMessage}, \texttt{detectedLanguage}, \texttt{normalizedMessageEn}, \texttt{staffMessage}, \texttt{category}, \texttt{status}, ainsi que les timestamps de suivi.

Le workflow de statut suit le cycle : PENDING → ASSIGNED → IN\_PROGRESS → RESOLVED → CONFIRMED.

\subsubsection{Pipeline IA combinée}

La route \texttt{/analyze} du service IA exécute trois étapes en un seul appel :
\begin{enumerate}
  \item \textbf{Détection de langue} : bibliothèque \texttt{langdetect} ;
  \item \textbf{Traduction} : modèle pré-entraîné NLLB-200-distilled-600M ;
  \item \textbf{Classification} : modèle entraîné Logistic Regression + TF-IDF.
\end{enumerate}

\subsubsection{Classification par apprentissage automatique}

Le classifieur a été entraîné sur un dataset hôtelier de $\sim$500 exemples répartis en 6 catégories : MAINTENANCE, HOUSEKEEPING, RECEPTION, RESTAURANT, COMPLAINT, OTHER.

\textbf{Approche :} pipeline scikit-learn combinant des features TF-IDF word et char n-grams avec un classifieur Logistic Regression.

\textbf{Évaluation :} 11 pipelines ont été testés. Le meilleur modèle retenu est \textbf{LR\_combined} :

\begin{table}[H]
\centering
\caption{Métriques du modèle de classification LR\_combined}
\label{tab:metriques_ml}
\begin{tabular}{|l|c|}
\hline
\textbf{Métrique} & \textbf{Valeur} \\
\hline
Accuracy (jeu de test) & 0,8595 ($\sim$86\%) \\
\hline
F1 macro & 0,8605 \\
\hline
F1 weighted & 0,8604 \\
\hline
CV F1 macro (validation croisée 5-fold) & 0,7702 $\pm$ 0,0450 \\
\hline
\end{tabular}
\end{table}

\begin{quote}
\textbf{Note de prudence :} La validation croisée présente un F1 macro moyen d'environ 77\%, ce qui montre que les résultats restent dépendants du jeu de données et doivent être interprétés avec prudence. Les limitations identifiées incluent : messages courts ou ambigus, confusion entre les catégories RECEPTION et RESTAURANT, et nécessité d'enrichir le dataset avec des données réelles de l'hôtel pour améliorer la généralisation du modèle.
\end{quote}

\subsection{Étude conceptuelle}

\subsubsection{Dashboard manager}

Le dashboard manager comprend :
\begin{itemize}
  \item Onglet \textbf{Réclamations} : liste filtrée par catégorie, assignation à un employé, transfert entre services, suivi du statut ;
  \item Onglet \textbf{Employés} : création, liste avec statut en ligne (\texttt{lastSeenAt}), disponibilité.
\end{itemize}

\subsubsection{Application Flutter mobile}

L'application Flutter est organisée en architecture feature-layered :
\begin{itemize}
  \item \textbf{Authentification} : login employé, stockage sécurisé du token JWT ;
  \item \textbf{Liste des tâches} : réclamations assignées avec statut et détails ;
  \item \textbf{Scanner QR} : scan entrée (\texttt{entryTime}) et scan sortie (\texttt{exitTime}) ;
  \item \textbf{Résultat} : saisie FIXED/NOT\_FIXED avec commentaire.
\end{itemize}
