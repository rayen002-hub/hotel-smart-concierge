\chapter{Release 1 : Socle système et gestion hôtelière}
\label{chap:release1}

\section*{Introduction}

Ce chapitre détaille la première release du projet, consacrée à la mise en place des fondations techniques et fonctionnelles du système. Cette release comprend trois sprints : l'authentification et la sécurité, la gestion des chambres et réservations avec les QR codes, et le check-in numérique avec les fiches voyageurs.

\subsection{Étude conceptuelle}

Le système d'authentification repose sur :
\begin{itemize}
  \item \textbf{JWT (JSON Web Token)} pour la gestion des sessions sans état ;
  \item \textbf{bcrypt} pour le hachage sécurisé des mots de passe ;
  \item \textbf{RBAC} avec 5 rôles : ADMIN, RECEPTIONIST, MAINTENANCE\_MANAGER, HOUSEKEEPING\_MANAGER, EMPLOYEE ;
  \item \textbf{Middlewares de sécurité} : Helmet (headers HTTP), express-rate-limit, CORS.
\end{itemize}

\subsection{Étude conceptuelle}

Le workflow de check-in comprend :
\begin{enumerate}
  \item Vérification du numéro de réservation ;
  \item Passage de la réservation en statut \texttt{CHECKED\_IN} ;
  \item Saisie des fiches voyageurs multiples (adultes + enfants) ;
  \item Suivi de la complétion (\texttt{checkinCompletionStatus} : NOT\_STARTED, PARTIAL, COMPLETED) ;
  \item Génération du QR code sécurisé pour le client ;
  \item Le client peut également compléter ses formulaires voyageurs depuis son téléphone.
\end{enumerate}
