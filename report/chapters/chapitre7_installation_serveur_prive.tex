\chapter{Installation cible sur serveur privé hôtelier}
\label{chap:installation}

\section*{Introduction}

Ce chapitre décrit l'architecture de déploiement cible du système Hotel Smart Concierge sur le serveur physique privé de l'hôtel. Il distingue clairement le déploiement temporaire de démonstration (utilisé pour la soutenance) de l'installation réelle prévue sur l'infrastructure interne de l'hôtel.

% =============================================================
\section{Objectif du déploiement cible}
\label{sec:objectif_deploiement}

L'objectif est de permettre à l'hôtel de fonctionner de manière \textbf{autonome et indépendante de tout service cloud public}. Tous les composants du système (backend, base de données, service IA, stockage de fichiers) sont conçus pour être installés sur un serveur physique privé situé dans les locaux de l'hôtel.

% =============================================================
\section{Architecture physique cible}
\label{sec:architecture_physique}

L'architecture cible repose sur un serveur physique unique ou un petit cluster, hébergeant :
\begin{itemize}
  \item \textbf{Node.js + Express} : backend API ;
  \item \textbf{PostgreSQL} : base de données relationnelle ;
  \item \textbf{Python + FastAPI} : service d'intelligence artificielle ;
  \item \textbf{Nginx} : reverse proxy et serveur de fichiers statiques (frontend React build) ;
  \item \textbf{Stockage local} : remplacement de Supabase Storage par un stockage de fichiers local.
\end{itemize}

\section{Réseau local hôtelier et accès staff}
\label{sec:reseau_local}

Le système est accessible sur le réseau local de l'hôtel :
\begin{itemize}
  \item Les dashboards du personnel sont accessibles depuis les postes de travail connectés au réseau local ;
  \item L'espace client PWA est accessible via le Wi-Fi de l'hôtel après scan du QR code ;
  \item L'application Flutter se connecte au backend via le réseau local.
\end{itemize}

% =============================================================
\section{Base de données et stockage interne}
\label{sec:bdd_stockage}

PostgreSQL est le SGBD principal du système. Dans le déploiement cible réel, la base de données est installée localement sur le serveur de l'hôtel. Le stockage des images (événements, etc.) est assuré par un répertoire local du serveur, remplaçant Supabase Storage utilisé dans l'environnement de démonstration.

% =============================================================
\section{Sécurité et confidentialité}
\label{sec:securite}

\begin{itemize}
  \item Toutes les communications internes sont sur le réseau local de l'hôtel ;
  \item Les données sensibles (passeports) sont chiffrées en base de données ;
  \item Les secrets (JWT secret, credentials DB) sont stockés dans des variables d'environnement ;
  \item L'accès externe peut être limité ou désactivé selon la politique de l'hôtel.
\end{itemize}

% =============================================================
\section{Sauvegarde et maintenance}
\label{sec:sauvegarde}

\begin{itemize}
  \item \textbf{Base de données} : sauvegarde régulière via \texttt{pg\_dump} ;
  \item \textbf{Fichiers} : sauvegarde du répertoire de stockage local ;
  \item \textbf{Mise à jour} : procédure de mise à jour du code via \texttt{git pull} et redémarrage des services.
\end{itemize}

\section{Déploiement temporaire de démonstration}
\label{sec:deploiement_demo}

Pour les besoins de la démonstration et de la soutenance, le système a été déployé temporairement sur des plateformes cloud publiques :

\begin{table}[H]
\centering
\caption{Plateformes de déploiement temporaire (démonstration uniquement)}
\label{tab:deploiement_demo}
\begin{tabular}{|l|p{8cm}|}
\hline
\textbf{Plateforme} & \textbf{Usage temporaire} \\
\hline
Supabase & Hébergement PostgreSQL de démonstration + stockage d'images \\
\hline
Vercel & Hébergement temporaire de la web-app (frontend React/PWA) \\
\hline
Render & Hébergement temporaire du backend Node.js/Express \\
\hline
Hugging Face Spaces & Hébergement temporaire du service IA FastAPI (avec Docker) \\
\hline
\end{tabular}
\end{table}

\begin{quote}
\textbf{Important :} Vercel, Render, Hugging Face Spaces et Supabase ont été utilisés \textbf{uniquement pour le déploiement temporaire de démonstration et de soutenance}. Le déploiement cible réel est prévu sur un \textbf{serveur physique privé de l'hôtel}, sans dépendance au cloud public.
\end{quote}
